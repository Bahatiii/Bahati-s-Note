## VBTS:
Vision-Based Tactile Sensor，是一种触觉传感器，通过 摄像头 + 柔性 / 透明接触表面 来感知触摸。换句话说，当机器人接触物体时，这种传感器捕获接触表面的形变、压痕或标记移动，然后用视觉 (camera) 图像来进行触觉感知。

## 解决了什么问题：
基于视觉的触觉传感器由于其复杂的物理特性和高分辨率视觉信号处理要求，缺乏高效且准确的模拟工具，限制了触觉机器人研究的规模和范围。扩展配备vbts的机器人仿真主要挑战在于精确建模超弹性软凝胶垫及其接触（刚体近似、软体仿真）。

## 论文提出了：
taccel，一个用于扩展集成vbts的高性能仿真平台，结合精确物理建模、逼真触觉信号生成和大规模并行化，用于仿真机器人、触觉传感器和触觉信号生成。

通过结合先进的固体材料模拟技术IPC和ABD（增量式接触 仿射体动力学），并在GPU上高效实现，可以构建一个VBTS模拟平台，实现强大的并行化速度。

怎么解决：
1. 高精度物理建模（IPC + ABD）：
* 软体建模（Gel Pad）： 凝胶垫使用有限元方法（FEM）建模为新虎克（neo-Hookean）固体，这是一种能准确描述超弹性软凝胶的材料模型。
* 接触处理（IPC）： 使用 增量势接触（IPC）方法来处理凝胶垫与其他物体（或机器人）之间的复杂接触。IPC 能在接触过程中数学上保证无交集和无反转的物理状态，确保接触解的精确性。
* 刚体建模（ABD）： 机器人连杆和刚性物体使用 仿射体动力学（ABD）建模。ABD 提供了比传统刚体动力学更高效且精确的运动模拟，尤其是在处理碰撞和铰接时。
2. 大规模 GPU 加速（NVIDIA Warp）：
* Taccel 将整个 IPC-ABD 物理系统高效地实现在 NVIDIA Warp 框架上。Warp 是一个高性能的 Python 框架，旨在为 GPU 编程提供便利，它使得大规模的并行计算成为可能。
* 这种实现使得 Taccel 能够在单个 GPU 上同时模拟数千个环境，从而实现超高的加速比。
3. 高保真触觉信号生成：
* 深度图/法线提取： 首先，从 FEM 模拟得到的凝胶垫形变表面上提取精确的深度图 d 和法线图 n。
* RGB 图像生成： 遵循 SOTA 方法，使用一个深度神经网络（DNN）作为渲染模型。这个 DNN 经过训练，能够将深度信息和法线信息映射为逼真的 RGB 触觉图案，从而实现了 Sim-to-Real 视觉信号的高保真度。
